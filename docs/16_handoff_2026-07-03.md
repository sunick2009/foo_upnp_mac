## M3 的下一步（依 ADR-013，順序重要）

1. **Toolchain 風險驗證**: 用 CLT + CMake 試編一個最小 ObjC++ bundle
   （`add_library(... MODULE)` + Cocoa link），不要先裝完整 Xcode。
   本地磁碟不夠是已知限制，ADR-013 的策略就是繞開它。
2. **引入 foobar2000 macOS SDK**: 從 foobar2000.org 下載，不 commit 進 repo，
   授權保守處理，並驗證 SDK 能在 CMake + CLT 下編譯。
   若失敗，啟動 ADR-013 的 fallback（純 CI 編譯）。
3. **GitHub mirror + Actions macOS workflow**: `macos-14` runner 作為 CI 守門，
   GitLab 仍是主工作流。
4. 之後才是 component 本體: `ComponentEntry`、`Preferences`、Browser panel、
   playlist 整合（結構見 `docs/06_foobar2000_component_plan.md`，但 build 用 CMake，
   不用 `.xcodeproj`）。
